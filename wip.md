**Findings**

4. `P2` The calendar and adjacent-navigation landmarks are empty in the server-rendered HTML and only become meaningful after JS runs. That means no-JS users and some assistive/crawler contexts get an empty `section` and empty `nav`, which is poor progressive enhancement. At minimum, the adjacent links should be rendered in HTML first and enhanced with JS after; ideally the nav links should also be a list (`ul/li`) instead of bare anchors inside generic `div`s. See [src/layouts/meditation.njk](/Users/dylankenneally/git/language-of-letting-go/src/layouts/meditation.njk:9), [src/index.njk](/Users/dylankenneally/git/language-of-letting-go/src/index.njk:18), and [src/includes/adjacent-meditation-nav-script.njk](/Users/dylankenneally/git/language-of-letting-go/src/includes/adjacent-meditation-nav-script.njk:68).

**Optimisations**

The wrapper in [src/layouts/base.njk](/Users/dylankenneally/git/language-of-letting-go/src/layouts/base.njk:347) is a generic `<section>` without its own heading, so semantically it would be cleaner as a `<div>` unless you intend it to be a named region. The calendar `<section aria-label="Pick a Date">` is acceptable because it has an accessible name, but a visible heading like “Browse by date” would make that region clearer for everyone.

If you want, I can turn this review into a concrete patch and update the templates to:
1. promote page titles to `h1`,
2. switch dates to `<time>`,
3. use semantic quote markup,
4. add SSR fallback navigation with list semantics.
